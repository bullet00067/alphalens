# K線型態自動識別與決策支援系統開發計畫

## 階段一：型態定義與數學模型建立 (Pattern Definition & Math Modeling)
此階段的目標是將「視覺上的圖形」轉化為「程式可判斷的數學條件」。

### 1. 建立標準型態庫 (Pattern Library)
將市場上常見的型態進行分類，初期建議先實作勝率較高且幾何特徵明顯的基礎型態，後續再擴充：
* **突破/收斂型態 (Bilateral/Continuation):** 
    * 上升三角 (Ascending Triangle)
    * 下降三角 (Descending Triangle)
    * 對稱三角 (Symmetrical Triangle)
    * 箱型區間/矩形 (Rectangle / Consolidation)
    * 旗型 (Flag) / 楔型 (Wedge)
* **反轉型態 (Reversal):**
    * 雙重頂/底 (Double Top / Double Bottom - 即 M頭/W底)
    * 頭肩頂/底 (Head and Shoulders Top / Bottom)

### 2. 定義幾何判定規則 (Rule-based Heuristics)
利用 PIP 找出的高點集合 $H = \{H_1, H_2, ...\}$ 與低點集合 $L = \{L_1, L_2, ...\}$ 建立判定邏輯：
* **斜率計算 (Slope Analysis):** 計算相鄰高點連線（壓力線）與相鄰低點連線（支撐線）的斜率。
* **容差值設定 (Tolerance Threshold):** 現實股市不會有完美的水平線。需設定容差參數 $\epsilon$（例如價格差異在 1.5% 以內視為「等高」或「等低」）。
    * *範例：下降三角的數學特徵*
        * 支撐線條件：$L_1 \approx L_2 \approx L_3$ (斜率 $\approx 0 \pm \epsilon$)
        * 壓力線條件：$H_1 > H_2 > H_3$ (斜率 $< 0$)

## 階段二：核心演算法開發 (Core Algorithm Development)
將第一階段的數學模型實作成可高效執行的模組，由於使用者調整 Scale 時需動態重繪，效能是關鍵。

### 1. 滑動視窗掃描 (Sliding Window Algorithm)
* **動態擷取:** 根據使用者當下可見的 K 線範圍（Scale），取出該範圍內的 PIP 序列。
* **特徵分組:** 擷取最近的 $N$ 個 PIP 點（例如 $N=5$ 通常能構成一個完整的三角收斂或M頭），將其分為高點陣列與低點陣列。

### 2. 型態匹配引擎 (Pattern Matching Engine)
* **決策樹 / 規則引擎:** 將分離出的高低點陣列丟入各個型態的判定函式。
* **信心分數計算 (Confidence Score):** 若完美的水平線分數為 100，根據實際斜率與容差的偏差程度給予評分（例如 85 分的頭肩頂）。低於門檻（如 70 分）則不顯示，避免誤導。

### 3. 突破偵測 (Breakout Detection)
* **邊界穿越:** 判斷最新一根 K 線的收盤價是否穿越了由 PIP 點定義出來的「支撐/壓力邊界線」。
* **量能輔助:** 結合成交量 (Volume) 過濾假突破（突破時若帶量，可提高決策權重）。

## 階段三：視覺化與 UI/UX 實作 (Visualization & User Interface)
將後端算出的型態與決策資訊優雅地呈現在現有的深色圖表上。

### 1. 圖表繪製擴充
* **動態趨勢線:** 根據識別出的型態，自動畫出壓力線與支撐線（如虛線與實線）。
* **型態著色:** 可以在型態構成的區域內填入非常透明的背景色（例如透明度 10% 的黃色），讓使用者一眼看出區間範圍。

### 2. 決策支援資訊面板 (Decision Support Tooltips/Panel)
* **標籤顯示:** 在圖表適當位置（或懸浮視窗）顯示目前偵測到的型態名稱（例如：`偵測到：下降三角 (看跌)`）。
* **自動計算停損停利 (Auto SL/TP):** 
    * 根據古典技術分析，目標價通常等於「型態的垂直高度」。系統可據此動態給出 Target 1 的建議價格。
    * 停損價可自動設定在支撐線下方（看跌）或壓力線上方（看漲）一定比例。

## 階段四：回測與效能優化 (Backtesting & Optimization)

### 1. 歷史數據回測 (Historical Validation)
* 使用熟悉的工具（如 Python 結合 pandas/numpy）撰寫離線腳本。
* 將過去幾年的台股或美股日 K 資料餵給型態識別引擎，統計各種型態出現後的「突破成功率」，提供數據背書。

### 2. 效能調優
* **快取機制 (Caching):** 若使用者頻繁縮放/平移圖表，針對已計算過的歷史區段 PIP 與型態結果進行快取，避免重複計算。
* **Web Worker (若為 Web 端):** 將複雜的 PIP 降噪與型態匹配演算法放到背景執行緒計算，確保前端 K 線圖拖曳時的幀率 (FPS) 順暢。

## 階段五：進階功能與 AI 整合 (未來展望)
在基礎型態庫穩定運作後，可以考慮導入更進階的分析：
* **多週期共振:** 允許系統同時檢查不同時間維度 (15m, 60m, Day)，若日線與小時線同時出現看漲型態，則給予更強烈的買進訊號。
* **整合 LLM 解盤:** 將「當前價格、識別出的型態、PIP 壓力支撐位」打包成 Prompt 送給 LLM（例如 Gemini 或 Claude），讓 AI 自動生成一段給使用者的「盤勢解讀與操作建議」文字。
