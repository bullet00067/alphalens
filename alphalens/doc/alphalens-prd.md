# 產品需求文件 (PRD)：PIP 型態辨識與動態繪圖決策支援系統

## 1. 專案概述 (Overview)

本專案旨在升級現有的 K 線圖表分析工具。目前系統已能根據底層演算法給出買賣訊號（箭頭）、設定停損 (Stop Loss) 與目標價 (Target 1)，並提供多空信心指數 (Forecast Confidence)。
**下階段目標**：導入基於 PIP (Perceptually Important Points) 演算法的「幾何型態辨識」，將抽象的買賣訊號轉化為具備高度視覺解釋性的「突破型態圖」（如：下降三角形、區間整理、上升三角形），為使用者提供更直覺的決策支援。

## 2. 現狀與目標差異分析 (Gap Analysis)

| 功能面向 | 目前實作現狀 (As-Is) | 目標期望 (To-Be) |
| --- | --- | --- |
| **圖表視覺呈現** | 僅顯示買賣箭頭、單一軌跡線（黃線）、平行的停損與目標價位線。 | 需動態繪製出構成型態的**支撐線 (Support)** 與 **阻力線 (Resistance)**，包含傾斜的趨勢線（連接 Lower Highs 或 Higher Lows）。 |
| **轉折點標示** | 無明確標示局部高低點。 | 需基於 PIP 演算法，在圖上明確標示出構成型態的關鍵節點（LH: 較低的高點、HL: 較高的低點等）。 |
| **決策輔助資訊** | 僅提供籠統的 Bull/Bear 信心指數 (如 60%)。 | 需明確標示**當前識別出的型態名稱**（例如：「當前型態：上升三角形突破」），讓使用者知道訊號的邏輯來源。 |
| **資料準確度要求** | 未明示即時報價的驗證機制。 | **強制要求**：在產出任何型態與價位決策前，必須透過 Yahoo Finance 連結進行雙重驗證，確保計算用的股價與即時爬蟲取得的股價完全一致。 |

## 3. 核心功能需求 (Feature Requirements)

### Feature 1: 動態型態幾何繪圖 (Dynamic Pattern Rendering)

* **需求描述**：在使用者調整 K 線圖縮放比例 (Scale) 時，圖表需動態重繪構成當前型態的邊界線。
* **實作細節**：
* 接收後端或演算法算出的 PIP 序列。
* 在 K 線圖層之上新增一個繪圖層 (Overlay)。
* **水平線繪製**：若偵測到多個相近的局部高點/低點，畫出水平的「阻力線」或「支撐線」。
* **趨勢線繪製**：若偵測到遞減的高點 (Lower Highs) 或遞增的低點 (Higher Lows)，畫出傾斜的趨勢線。



### Feature 2: 型態特徵與突破點標註 (Pattern & Breakout Annotation)

* **需求描述**：在圖表上將抽象的數學特徵具象化，幫助使用者理解型態。
* **實作細節**：
* 在關鍵 K 線的上下方加入小文字標籤（例如圖中的 `LH` 或 `HL`）。
* 當最新價格 K 線實體 (Body) 穿越定義的支撐/阻力線時，在該突破點繪製明顯的「突破 (Breakout)」標示（可使用特定圖示或高光圈選）。



### Feature 3: 決策支援面板更新 (Decision Support Panel Update)

* **需求描述**：更新現有的 Forecast Confidence 區塊，加入型態學的解釋。
* **實作細節**：
* 新增欄位 `Detected Pattern`（偵測型態）：顯示文字如「上升三角形 (Ascending Triangle)」、「區間 (Rectangle)」。
* 停損 (Stop Loss) 與 目標價 (Target 1) 的計算邏輯，需與偵測到的型態連動（例如：突破區間後的目標價，通常為突破點加上區間的高度）。



### Feature 4: 資料雙重驗證機制 (Data Validation Mechanism) - **[高優先級]**

* **需求描述**：確保決策支援資訊的絕對準確性。
* **實作細節**：
* 在執行 PIP 演算法與產生分析報告/圖表前，系統必須先呼叫 Yahoo Finance API 或爬蟲模組。
* 比對系統內部暫存的最新 K 線收盤價與 Yahoo Finance 的即時報價。
* 若兩者一致，才渲染型態圖與給出決策；若不一致，需觸發資料更新重試或提示使用者資料同步中。



## 4. 驗收標準 (Acceptance Criteria)

1. **繪圖正確性**：當後端識別出「下降三角形」時，前端圖表必須正確畫出一條水平支撐線與一條向下的趨勢線，且線條精準連接對應的 PIP 點。
2. **RWD 與縮放支援**：使用者縮放 (Zoom in/out) K 線圖時，繪製的趨勢線與支撐/阻力線必須跟著 K 線正確縮放與平移，不會發生位移破圖。
3. **UI 資訊完整性**：除了現有的綠色與紅色箭頭，畫面上必須能看到目前的「型態名稱」。
4. **驗證邏輯通過**：系統日誌 (Log) 需證明在產生畫面標示前，已成功執行 Yahoo Finance 的雙重股價驗證。
